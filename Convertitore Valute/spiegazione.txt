Un socket è un punto di accesso (endpoint) per la comunicazione tra due processi, che possono essere sulla stessa macchina o su macchine diverse in rete.
È l’astrazione che permette a un programma di inviare e ricevere dati attraverso la rete.

Composizione di un socket (nei protocolli TCP/IP)
Un socket è identificato in modo univoco da una quadrupla (o coppia, a seconda del contesto):

Indirizzo IP del host (es. 192.168.1.1)

Numero di porta (es. 80, 443, 12345)

Protocollo di trasporto (TCP o UDP)

Dominio (es. AF_INET per IPv4, AF_INET6 per IPv6)

Nel codice, in Python, si crea con:

python
import socket
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)  # TCP
In C (come da tuo PDF), si usa una struttura sockaddr_in che contiene:

sin_family (es. AF_INET)

sin_port (porta, in network byte order)

sin_addr (indirizzo IP)

🧵 Cos’è un thread?
Un thread (filo di esecuzione) è la più piccola unità di esecuzione all’interno di un processo.
Un processo può contenere più thread che condividono lo stesso spazio di memoria e risorse (variabili globali, file descriptor, ecc.), ma ognuno esegue un flusso di istruzioni indipendente.

Perché si usano nei server?
Per gestire più client contemporaneamente senza dover attendere che uno finisca prima di servire il successivo. Ogni connessione viene assegnata a un thread separato.

Esempio in Python:

python
import threading
thread = threading.Thread(target=funzione, args=(conn, addr))
thread.start()
Attenzione: se più thread accedono a dati condivisi (es. contatore delle prenotazioni), bisogna usare un lock (threading.Lock()) per evitare race condition.

📡 Cos’è un protocollo? Differenza tra TCP e UDP
Protocollo
Un protocollo è un insieme di regole che due entità (es. computer) devono seguire per scambiare dati in modo corretto e comprensibile. Definisce formati, sequenze di messaggi, gestione errori, ecc.

TCP (Transmission Control Protocol)
Orientato alla connessione: prima di scambiare dati, si stabilisce una connessione (three‑way handshake).

Affidabile: garantisce che i dati arrivino in ordine, senza errori, duplicati o perdite (ritrasmissioni).

Controllo di flusso e congestione.

Più lento e con overhead maggiore.

Usato per: web (HTTP), email (SMTP), file transfer (FTP), ecc.

UDP (User Datagram Protocol)
Senza connessione: si inviano datagrammi indipendenti, senza stabilire un canale.

Non affidabile: i pacchetti possono perdersi, arrivare duplicati o fuori ordine; nessuna ritrasmissione.

Leggero e veloce.

Usato per: streaming video/audio, DNS, giochi online in tempo reale, VoIP.

Caratteristica	TCP	UDP
Connessione	Sì	No
Affidabilità	Alta (con ritrasmissioni)	Bassa (best effort)
Ordinamento	Sì	No
Velocità	Più bassa	Più alta
Overhead	Maggiore	Minore
✉️ Differenza tra send() e sendall() in Python
Entrambi inviano dati attraverso un socket, ma:

send(bytes)
Tenta di inviare i dati, ma potrebbe inviarne solo una parte (restituisce il numero di byte effettivamente inviati).
Va usato in un ciclo per garantire l’invio completo.

sendall(bytes)
Continua a inviare automaticamente finché tutti i dati non sono stati trasmessi o si verifica un errore.
Più comodo e usato nella maggior parte dei casi.

Esempio:

python
# send: potrebbe inviare solo 500 byte su 1024
inviato = sock.send(data)

# sendall: garantisce invio completo (o eccezione)
sock.sendall(data)
📥 Cosa fa recv(1024)?
recv(buflen) riceve dati da un socket.

L’argomento 1024 è la dimensione massima del buffer (in byte) che si è disposti a ricevere in una volta.

Restituisce i byte ricevuti (come oggetto bytes).

Se la connessione è chiusa dall’altro capo, restituisce b'' (stringa vuota).

Non significa che si ricevono esattamente 1024 byte; si ricevono al più 1024 byte. Se il mittente ha inviato meno, si riceve quella quantità minore. Per ricevere un messaggio lungo, spesso si chiama recv() in un ciclo finché non si riceve il dato atteso.

🔄 Cosa fanno encode() e decode()?
I socket lavorano con byte, non con stringhe.
Le stringhe in Python (tipo str) sono sequenze di caratteri Unicode; i byte (bytes) sono sequenze di valori 0–255.

encode() → converte una stringa in bytes, usando un encoding (di default UTF-8).
Esempio: "ciao".encode() → b'ciao'

decode() → converte bytes in stringa.
Esempio: b'ciao'.decode() → "ciao"

Nel codice di rete:

python
# Invio
conn.sendall(messaggio.encode())   # stringa → bytes

# Ricezione
dati = conn.recv(1024)
testo = dati.decode()              # bytes → stringa
